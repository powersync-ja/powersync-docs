PowerSync currently provides DevTools integrations for Dart/Flutter and Nuxt. These embed a diagnostics panel into your existing IDE or framework tooling, so you can inspect your app's local database, sync stream state, and credentials without launching a separate tool. They are available in development builds only.

<Info>
Support for additional frameworks is under consideration. If you'd like to see a specific integration, vote for it on the [PowerSync roadmap](https://roadmap.powersync.com).
</Info>

## DevTools vs. Sync Diagnostics Client

The DevTools integrations and the [Sync Diagnostics Client](/tools/diagnostics-client) serve different purposes and complement each other.

The DevTools integrations connect directly to your app's running process. This means they can surface client-side problems that the Sync Diagnostics Client cannot, such as pending local writes that are preventing sync from progressing, or a local checkpoint that is waiting to be applied. They also include a button to open the Sync Diagnostics Client pre-configured with the current user's endpoint and token, so you can jump to server-side inspection without generating a separate token.

The Sync Diagnostics Client connects to the PowerSync Service itself. Use it to verify that your service is replicating data correctly from your source database, or to inspect sync configuration and server-side state.

| | DevTools integrations | Sync Diagnostics Client |
|---|---|---|
| **What it connects to** | Your app's running process | The PowerSync Service instance |
| **Surfaces** | Pending writes, local sync state, stream status, local data | Replication state, sync configuration, server-side logs |
| **Environment** | Development only | Any environment |
| **SDK-specific** | Yes (Dart/Flutter, Nuxt) | No — works with all SDKs |
